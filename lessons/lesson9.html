Dynamic Memory

 Introduction

In this lesson you will learn about dynamic memory. Up until now, all of the space allocated in memory that you have been learning is known as static allocation. Static allocation is useful because it cleans up after itself when the program exits, but is not as versatile as dynamic memory. For example, if one were to allocate 30 bytes for a name space and a user enters a name more than 30 characters, there would not be enough space to contain their name. This can be mended with the use of dynamic memory. With dynamic memory, the namespace would be able to extend dynamically to the needs of the program, without extending into space that is harmful.

 Dynamic Memory Functions

In order to utilize dynamic memory in your program, you must first include the stdlib.h library, as you have previously learned from the C Library lesson. This library provides the required functions for dynamic memory, and it is a staple within the C programming community.

 Malloc

Malloc is the main memory allocation function. It applies to most circumstances and can easily be called upon any data type desired.

To put it (not so)simply, Malloc will allocate memory through the use of void pointers. These pointers can be cast to any data type that your program requires with the use of the right syntaxing. Further extending this, the arguments for malloc require the size of the data type you are attempting to allocate multiplied by the number of elements of that specific data type Due to the memory being dynamic, it is perfectly valid to only allocate enough space for one variable so you do not use up more than you need. In addition to just allocating space for one item, it can be allocated for multiple elements at once.
