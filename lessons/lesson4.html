Debugging and Troubleshooting

Introduction

In this section you will learn the importance of debugging and troubleshooting code in C. There are a multitude of tools to support debugging, such as the use of an IDE and other third party software. A widely accepted debugging tool in the programming community is called GDB.

  • Official GDB Website

    • GDB

    GDB is a powerful tool that allows you to debug code while it is being executed. Some of the main features include:

      Obtaining the values of variables at any given instance

      Call functions and reassign variables independent of the program's instructions

      Identifying the location in which bugs or crashes occur

    There are many useful commands to learn in GDB which can all be found through the official documentation. Additionally, the manual page ( man gdb ) can be called upon from the terminal for quick documentation on commands and their uses.

    Debugging Mentality

    Even with many tools available to support debugging, it does not neccesarily provide the required skills to be effective at eliminating bugs. Like any other tool, it depends on the competence of the user and how that tool is being utilized.

    Whatever your approach to debugging is, it is important to break down your code into identifable segments to easily distinguish the functional code and the areas in which potential bugs occur.
